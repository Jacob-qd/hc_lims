## 跨模块一致性问题

### 🔴 编号规则不统一

| 实体 | 当前编号格式 | 问题 |
|------|-----------|------|
| 委托 | ORD-YYYYMMDD-NNN | ✅ 合理 |
| 样品 | SMP20240521001 | ❌ 无分隔符, 不直观 |
| 任务 | TK-2025-NNN | ✅ 合理 |
| 报告 | RPT-YYYYMMDD-NNN | ✅ 合理 |
| COC链 | 未定义 | ❌ 缺编号规则 |
| ELN记录 | 未定义 | ❌ 缺编号规则 |
| 质控方案 | 未定义 | ❌ 缺编号规则 |

**修正**: 统一格式 `PREFIX-YYYYMMDD-NNN` (PREFIX=实体缩写, 日期=创建日期, NNN=3位流水)

### 🔴 时间格式不统一

| 位置 | 格式 | 
|------|------|
| 委托创建时间 | `2026-05-16 09:15` |
| 样品接收时间 | `2024-05-21 09:15` |
| 委托截止日期 | `2026-05-21` (仅日期) |
| 任务开始时间 | `2024-05-20` (仅日期) |
| Mock数据日期 | 混用 2024/2025/2026 |

**修正**: 全部统一为 `YYYY-MM-DD HH:mm`, 仅日期字段使用 `YYYY-MM-DD`

### 🔴 状态值不跨模块联动

| 场景 | 当前 | 问题 |
|------|------|------|
| 委托状态→样品状态 | 无关联 | 委托「已收样」时, 样品应自动变「已接收」 |
| 样品状态→任务状态 | 无关联 | 样品「检测中」时, 任务应已创建 |
| 任务完成→报告状态 | 部分关联 | 任务完成后报告应自动「可生成」 |
| QC违规→样品冻结 | 无关联 | QC违规应自动冻结关联样品 |

**修正**: 增加状态联动文档, 明确触发条件和传播规则

## 21 CFR Part 11 / CNAS 合规审查

### 🔴 审计日志完整性

| 操作 | 当前是否记录 | 修正 |
|------|:---:|------|
| 委托创建 | ❌ | 必须记录 |
| 委托变更 | ❌ | 必须记录 before/after |
| 样品接收 | ❌ | 必须记录接收人/时间/异常标记 |
| 结果录入 | ⚠️ localStorage | 必须记录每次修改的前后值 |
| 报告签发 | ✅ 有签名审计 | - |
| 权限变更 | ❌ | 必须记录 |

### 🔴 电子签名缺失场景

| 需要签名 | 当前 | 修正 |
|---------|:---:|------|
| 委托确认 | ❌ | 委托提交需客户或业务员签名 |
| 样品接收 | ❌ | 收样交接需双方签名 |
| 留样处置 | ❌ | 处置需签名确认 |
| 报告编制 | ✅ | - |
| 报告签发 | ✅ | - |
| 质控违规处置 | ❌ | CAPA关闭需QA签名 |

### 🔴 数据完整性 (ALCOA+)

| 原则 | 当前 | 问题 |
|------|:---:|------|
| Attributable (可归属) | ⚠️ | 部分操作无操作人记录 |
| Legible (清晰) | ✅ | - |
| Contemporaneous (同步) | ⚠️ | 允许事后补录, 无时间校验 |
| Original (原始) | ❌ | ELN 无"原始数据不可修改"保护 |
| Accurate (准确) | ✅ | - |
| Complete (完整) | ❌ | 删除操作无审计记录 |
| Consistent (一致) | ❌ | 多处数据不同步 |
| Enduring (持久) | ❌ | localStorage 存储易丢失 |
| Available (可用) | ⚠️ | 数据归档未实现 |

## 真实用户场景模拟

### 场景1: "月底突击 — 30个委托, 200个样品, 50份报告"

```
Day 1: 业务员接到客户电话→创建30个委托→Excel批量导入200个样品
  ❌ 当前: 无Excel批量导入样品功能(设计文档有, 未实现)

Day 3: 收样员接收→部分温度超标(标记有条件接收)→分样
  ❌ 当前: 无异常接收流程, 无分样

Day 5: 检测完成→批量生成50份报告→批量审核→批量签发
  ⚠️ 批量签发已实现, 批量审核未实现

Day 7: 客户投诉→发现3份报告有误→作废重签→通知客户
  ❌ 当前: 无作废重签流程
```

```
3. 张伟的资质授权和培训记录

4. 过去3年的审计日志
```

### 场景3: "系统故障 — 检测员正在录入结果时断网"

