# 模块 3: 报告管理 — 竞品调研 + 设计 + 实现对比

---

## 1. 竞品调研

### 1.1 金现代 (JXD)

金现代报告管理核心能力：
- **一键出报告**：检测完成 → 自动匹配模板 → 自动填充数据 → 生成报告草稿
- **报告模板引擎**：可自定义模板 (页眉/页脚/表格/图表/签名位)，支持 Word/PDF 双格式
- **多级审核**：编制 → 技术审核 → 批准 → 终签，每级可退回
- **电子签章**：支持国密 SM2 签名，签章位置可配，签后 PDF 固化防篡改
- **批量签发**：多选 → 批量审核 → 批量签名
- **报告自动分发**：签发后自动推送客户门户 + 邮件通知 + 监管报送
- **报告作废/重签**：作废原报告 + 新建版本 + 保留完整关联

> 来源：JXD 官网 + 化工方案 + 环境监测方案

### 1.2 Thermo Fisher SampleManager

SampleManager 报告特点：
- **Report Designer**：可视化报告设计器，支持拖拽图表/表格/图片
- **Template Versioning**：报告模板版本管理，历史报告保留旧模板
- **Multi-language Reports**：同一数据源生成多语言报告
- **Scheduled Reports**：定时生成 (如每周一自动生成上周 QC 报告)

### 1.3 LabWare

LabWare 报告特点：
- **Crystal Reports 集成**：支持 Crystal Reports 模板
- **CoA (Certificate of Analysis)**：分析证书专用模板
- **Report Queue**：报告生成队列，支持批量生成和优先级

### 1.4 STARLIMS

STARLIMS 报告特点：
- **Advanced Reporting**：内置 BI 报表引擎
- **Regulatory Submission**：支持 FDA/EMA 等监管格式
- **Watermark & Security**：动态水印 (查看人/时间/IP)，防截屏

### 1.5 三维天地

三维天地报告特点：
- **CMA/CNAS 合规报告格式**：自动生成符合评审要求的报告
- **报告防伪**：验真二维码 + 电子签章 + PDF 签名
- **客户自助下载**：客户门户扫码下载验证

### 1.6 最佳实践总结

| 能力 | 推荐方案 | 来源 |
|------|---------|------|
| 报告生成 | 自动匹配模板 + 自动填充数据 | JXD |
| 审核流程 | 编制→技术审核→批准→终签 (可退回) | JXD + LabWare |
| 电子签名 | SM2 国密 + 签名含义 + 签后固化 | JXD |
| 批量操作 | 批量审核 + 批量签发 | JXD |
| 模板管理 | 版本化 + 历史报告保留旧模板 | SampleManager |
| 自动分发 | 签发 → 客户门户 + 邮件 + 监管 | JXD + STARLIMS |
| 报告作废 | 作废 + 新建版本 + 关联保留 | JXD |
| 防伪 | 验真二维码 + 动态水印 + PDF 签名 | 三维天地 + STARLIMS |

## 2. User Story

### US1 — 一键生成报告
> 作为**报告编制人**，样品检测完成后，我点击「生成报告」，系统自动匹配该样品类型和检测项目的报告模板，填入所有检测结果、方法、限值和判定，生成草稿。

### US2 — 报告在线编辑
> 作为**报告编制人**，生成的报告草稿应支持在线编辑——修改结论文字、调整图表位置、添加备注。

### US3 — 多级审核退回
> 作为**审核人**，我发现报告中的图表单位错误，可以退回给编制人并填写退回原因。编制人修改后重新提交，系统保留版本差异。

### US4 — 批量签发
> 作为**授权签字人**，月底有 50 份报告需要签发。我应能多选后一键批量签名，而不是逐份签。

### US5 — 报告作废重签
> 作为**授权签字人**，发现已签发报告有误，我应能作废旧版本并生成新版本，系统保留作废原因和关联关系。

### US6 — 客户自助下载
> 作为**客户**，报告签发后我收到短信通知，点击链接输入验证码即可下载 PDF 并验证真伪。

## 3. 功能设计

### 3.1 报告全生命周期状态机

```
检测完成 → 生成草稿 → 编制中 → 提交技术审核
                                    ↓
                              审核通过 → 提交批准
                              审核退回 → 编制中 (版本+1)
                                    ↓
                              批准通过 → 待签发
                              批准退回 → 编制中
                                    ↓
                              电子签名 → 已签发 (PDF固化)
                                    ↓
                              客户下载 / 监管报送
                                    ↓
                          [如需修改] 作废 → 新建版本 (v2)
```

### 3.2 报告操作权限矩阵

| 操作 | 编制人 | 审核人 | 授权签字人 | QA | 客户 |
|------|:---:|:---:|:---:|:---:|:---:|
| 生成草稿 | ✅ | ✅ | ✅ | - | - |
| 编辑报告 | ✅(自己的) | ❌ | ❌ | - | - |
| 提交审核 | ✅ | - | - | - | - |
| 审核通过/退回 | - | ✅ | - | - | - |
| 批准 | - | - | ✅ | - | - |
| 电子签名 | - | - | ✅ | - | - |
| 作废重签 | - | - | ✅ | - | - |
| 查看报告 | ✅(自己的) | ✅ | ✅ | ✅(全部) | ✅(自己的) |
| 下载 PDF | - | - | - | - | ✅(自己的) |

### 3.3 批量签发工作流

```
列表页多选 (勾选 checkbox)
    ↓
点击「批量签发」
    ↓
预览弹窗：展示选中的报告列表 (编号/样品/结论)
    ↓
输入签名密码 + 选择签名含义
    ↓
确认 → SM2 签名 → PDF 固化
    ↓
进度条显示签发进度 (5/50... 50/50)
    ↓
完成 → 显示成功/失败明细
```

## 5. 开发实现规格 — 作废重签与自动分发 (最大缺失)

### 5.1 报告作废重签
```
ReportsPage → 已签发报告 → 点击[作废]:
├── 作废Modal:
│   ├── 当前报告信息
│   ├── 作废原因 (必填, TextArea)
│   ├── 是否创建新版本 (Checkbox, 默认勾选)
│   └── [确认作废]
├── 确认后:
│   ├── 原报告 status → 'voided'
│   ├── 新增 voidedAt, voidedBy, voidReason
│   └── 如勾选创建新版本:
│       ├── 复制原报告数据 →
│       ├── 新 reportNo → 原号-V2
│       ├── 新 status → 'draft'
│       └── 关联 voidedFrom: 原reportId
```

### 5.2 自动分发
```
报告签发后:
├── 客户门户: 报告状态变为「可下载」
├── 邮件通知: 发送到客户邮箱 (含下载链接)
├── 短信通知: (可选) 发送到客户手机
└── 监管报送: (可选) 自动推送到监管平台API
```

### 5.3 数据模型
```typescript
interface ReportVoid {
  reportId: string;
  voidReason: string;
  voidedBy: string;
  voidedAt: string;
  createNewVersion: boolean;
  newReportId?: string;
}

interface ReportDistribution {
  reportId: string;
  channels: ('portal' | 'email' | 'sms' | 'regulatory')[];
  emailTo?: string;
  smsTo?: string;
  distributedAt: string;
  status: 'pending' | 'sent' | 'failed';
}
```
