## 1. 竞品调研

### 1.1 金现代 (JXD)

金现代检测管理核心能力：
- **任务自动拆分**：样品登记 → 按检测项目自动拆分为独立任务
- **设备自动排配**：考虑设备通道/预约/维护/校准状态自动排程
- **任务状态流转**：待接收→待执行→执行中→待复核→待审核→已签发
- **原始记录模式**：ELN 模板驱动，支持自动取数 + 手工观察双模式
- **平行样/加标**：系统自动在样品序列中插入质控样

### 1.2 Thermo Fisher SampleManager

- **Worklist**：按仪器生成工作列表，批量上机
- **Result Entry by Exception**：只录入异常结果，正常结果自动通过
- **Automatic Re-test**：超标结果自动创建复测任务

### 1.3 LabWare

- **Batch Management**：批次管理，同一批次样品关联质控
- **Stability Testing**：稳定性考察，按时间点自动生成检测任务

### 1.4 STARLIMS

- **Result Lifecycle**：结果录入→复核→批准→发布 完整状态机
- **Audit Trail on Results**：每次结果修改记录前后值、原因、操作人

### 1.5 最佳实践

| 能力 | 推荐方案 | 来源 |
|------|---------|------|
| 任务拆分 | 自动按检测项目拆分 | JXD |
| 设备排程 | 可视化日历 + 冲突检测 | JXD |
| 结果录入 | ELN 模板 + 自动取数 + 手工双模式 | JXD |
| 复测管理 | 超标自动创建复测任务 | SampleManager |
| 批次管理 | 批次关联质控 + 统一判定 | LabWare |

## 2. User Story

### US1 — 任务自动生成
> 样品接收完成后，系统自动按检测项目拆分为独立任务，每个任务关联方法、仪器和检测员。

### US2 — 结果录入（ELN 驱动）
> 打开任务 → 加载 ELN 模板 → 自动带入样品信息 → 填写实验数据 → 系统自动计算和判定 → 提交复核。

### US3 — 超标自动复测
> COD 结果 125 mg/L（标准 ≤50），系统自动标记"超标"，创建复测任务并通知检测员。

### US4 — 批量结果录入
> 同一批 20 个 COD 样品，支持类似 Excel 的批量填写模式，一次提交。

## 4. 开发实现规格 — 复测管理 (最大缺失)

### 4.1 超标自动复测流程
```
结果判定"超标" →
├── 自动标记任务 status = 'retest_required'
├── 创建复测任务:
│   ├── 复制原任务信息 (样品/方法/检测项目)
│   ├── 关联原任务 (retestOf: originalTaskId)
│   ├── 优先级提升到 'urgent'
│   └── 通知检测员 + QA
└── 原任务保留, 不自动归档
```

### 4.2 复测结果处理
```
复测任务完成 →
├── 复测结果符合 → 原任务判定修正 + 备注"复测后修正"
├── 复测结果超标 → 升级为 OOS → 自动创建偏差记录
└── QA 审核确认
```

### 4.3 数据模型
```typescript
interface Task {
  // ... 原有字段 ...
  retestOf?: string;           // 原超标任务ID
  retestReason?: string;       // "初次检测超标: COD=125 (>50)"
  originalResult?: string;     // 原始结果值
  retestCount: number;         // 复测次数 (最多3次)
}
```
