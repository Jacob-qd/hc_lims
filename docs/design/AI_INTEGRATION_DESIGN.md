# HC-LIMS AI 集成设计方案 v2.0

> 目标：将 AI 从"独立聊天窗口"升级为"贯穿检测全流程的智能助手"，在关键业务节点提供实时决策支持。

---

## 1. 调研结论

### 1.1 行业痛点
| 环节 | 痛点 | AI 解决方向 |
|------|------|------------|
| 样品登记 | 手工录入效率低、信息不完整 | OCR + 智能补全 |
| 任务分配 | 依赖人工经验，排程不合理 | 负荷感知 + 智能推荐 |
| 结果录入 | 异常值难发现，误判风险高 | 实时异常检测 |
| 报告编制 | 格式固定但撰写耗时 | 自动生成摘要/结论 |
| 质控监控 | 事后发现失控，响应滞后 | 预测性质控预警 |
| 客户服务 | 重复咨询多，响应慢 | AI 客服 + 进度查询 |

### 1.2 技术选型
- **大模型**：支持接入 OpenAI / 通义千问 / 本地模型（统一抽象层）
- **向量数据库**：检测知识库 RAG（方法标准、判定规则）
- **实时推理**：轻量模型本地运行（异常检测、分类）
- **工作流编排**：LangChain 风格 Chain（规划→执行→验证）

## 2. User Stories

### US-AI-01 样品智能登记
> 作为**收样员**，我拍照上传委托单，系统自动识别客户信息、样品类型、检测项目，减少手工录入。

**验收标准：**
- 支持上传图片/PDF，OCR 识别客户名称、样品编号、检测项目
- 识别准确率 > 85%（mock 演示）
- 识别结果可编辑确认后一键创建样品

### US-AI-02 智能任务分配
> 作为**实验室主管**，我点击"智能分配"，系统根据人员当前负荷、技能匹配度、仪器可用性推荐最佳分析员。

**验收标准：**
- 展示人员负荷热力图（当前待检测任务数）
- 推荐逻辑：技能匹配 + 负荷最低 + 仪器就近
- 一键采纳推荐，自动创建任务分配记录

### US-AI-03 结果异常实时预警
> 作为**检测员**，我录入结果时，如果数值偏离历史均值或超出方法线性范围，系统立即黄色/红色警告。

**验收标准：**
- 输入框失焦时触发 AI 校验
- 异常规则：超方法检出限 10 倍、超历史均值 ±3σ、平行样 RPD 超限
- 提供"查看历史数据"快捷入口

### US-AI-04 报告智能生成
> 作为**报告编制员**，我点击"AI 生成摘要"，系统自动根据检测结果撰写"检测结论"和"建议"。

**验收标准：**
- 根据所有检测项的合格/不合格情况生成总结性文字
- 不合格项自动引用对应标准限值
- 生成内容可编辑，支持重新生成

### US-AI-05 预测性质控监控
> 作为**质量主管**，我在质控看板看到 AI 预测的"未来 7 天失控概率"，提前介入高风险项目。

**验收标准：**
- 基于历史质控数据训练的趋势预测
- 高风险项目（失控概率 > 30%）红色标记
- 给出具体建议（如：该换标准品了）

### US-AI-06 AI 流程助手（Copilot）
> 作为**任意角色**，我在页面右下角看到 AI Copilot，可随时询问"这个样品状态是什么？""帮我总结本周工作"。

**验收标准：**
- 全局悬浮按钮，支持展开/收起
- 上下文感知：根据当前页面提供相关快捷提问
- 支持执行操作："帮我把这个任务分配给李明"

## 4. API 设计

```
POST   /api/v1/ai/copilot         # 通用对话（上下文感知）
POST   /api/v1/ai/ocr             # OCR 识别图片
POST   /api/v1/ai/suggest-tasks   # 智能任务分配建议
POST   /api/v1/ai/check-anomaly   # 结果异常检测
POST   /api/v1/ai/report-draft    # 生成报告摘要
GET    /api/v1/ai/qc-prediction   # 质控趋势预测
GET    /api/v1/ai/knowledge       # 知识库查询（RAG）
```
