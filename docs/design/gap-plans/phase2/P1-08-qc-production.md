# P1-08 质控数据生产化

> 对标: StarLIMS QC Management | LabWare SPC
> 版本: v1.0 | 优先级: P1 | 估算: 2周

---

## 1. 功能概述

将当前 Mock 质控数据替换为真实数据源，扩展 Westgard 规则集，实现生产级质量控制能力。

### 核心功能

- 质控样品/质控测试数据模型
- 检测流程中插入质控样品
- 真实数据驱动的 L-J 控制图
- Westgard 多规则自动判定（扩展到完整规则集）
- QC 报告（月度/季度）

---

## 2. 功能需求

### FR-01: 质控数据模型

| 实体 | 字段 |
|------|------|
| QC 样品 | 名称、批号、浓度、有效期、基体、保存条件 |
| QC 测试 | QC样品ID、检测项ID、日期、结果、操作人、仪器 |
| QC 规则 | 规则类型、参数（均值、SD、控制限） |

### FR-02: Westgard 规则（扩展）

| 规则 | 说明 | 动作 |
|------|------|------|
| 1₂s | 1个值超出 2SD | ⚠️ 警告 |
| 1₃s | 1个值超出 3SD | 🔴 失控 |
| 2₂s | 连续2个值在同侧超出 2SD | 🔴 失控 |
| R₄s | 2个值差异超出 4SD | 🔴 失控 |
| 4₁s | 连续4个值在同侧超出 1SD | 🔴 失控 |
| 10x | 连续10个值在均值同侧 | ⚠️ 系统性偏差 |
| 8x | 连续8个值在均值同侧 | 🔴 失控 |

### FR-03: 控制图

- X̄ 图 (均值控制图)
- R 图 (极差控制图)
- S 图 (标准差控制图)
- 自动计算控制限 (±1SD, ±2SD, ±3SD)

---

## 3. API 设计

```
POST   /api/v1/qc/samples                 # 创建 QC 样品
POST   /api/v1/qc/tests                   # 录入 QC 测试结果
GET    /api/v1/qc/charts/:qcSampleId      # 获取控制图数据
GET    /api/v1/qc/rules                    # 规则配置
PUT    /api/v1/qc/rules/:id               # 更新规则参数
GET    /api/v1/qc/reports                  # QC 报告
POST   /api/v1/qc/verify                  # 校验最新数据
```
