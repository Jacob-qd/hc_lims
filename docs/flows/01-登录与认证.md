# 登录与认证 — 用户使用流程

**设计图**: `design/` (登录页无独立设计图，参考当前实现)
**路由**: `/login`
**状态**: ✅ 已实现 (145行)

---

## 主流程

```
[用户打开系统] → [未认证，RouteGuard重定向到 /login]
    → 输入用户名、密码、选择角色
    → POST /api/v1/auth/login
    → 成功 → 存储 token → 跳转 /dashboard
    → 失败 → 显示错误提示
```

### 步骤详情

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 访问系统首页 `/` | 未认证时自动跳转 `/login` |
| 2 | 输入 `username` | 表单校验：必填 |
| 3 | 输入 `password` | 表单校验：必填 |
| 4 | 选择 `role`（管理员/检测员/审核员） | 角色下拉选择 |
| 5 | 点击「登录」 | 按钮 loading 状态 |
| 6 | 系统验证 | 成功 → 跳转 Dashboard |
| 7 | 系统验证 | 失败 → message.error 提示 |
