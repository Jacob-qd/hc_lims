## 0. 竞品调研 — Thermo Fisher 赛默飞 ELN

### 0.1 产品定位

赛默飞（Thermo Fisher Scientific）是全球实验室信息化领导者，其 ELN 功能主要内置于 **Thermo Scientific™ SampleManager LIMS** 中，另外在生物分析领域有独立的 **Watson LIMS** 配套 ELN。赛默飞 ELN 的核心定位是**"结构化数据捕获 + 仪器集成 + 合规驱动"**，与自由格式的纸质替代型 ELN 有本质区别。

### 0.2 核心功能矩阵

| 功能维度 | Thermo Fisher ELN 特性 | HC-LIMS 现状 | 差距评估 |
|---------|----------------------|-------------|---------|
| **模板引擎** | 可视化模板设计器，支持字段类型：文本/数值/下拉/复选/表格/公式/仪器字段/条码字段 | 模板选择 + 基础字段映射 | ⚠️ 中 — 缺少可视化模板设计器 |

| **计算引擎** | 内置公式编辑器，支持跨字段计算、统计函数（均值/SD/RSD/线性回归）、单位自动换算 | 无内置计算 | 🔴 大 — 需补充计算能力 |
| **样品追溯** | 与 LIMS 样品管理无缝集成，实验记录可直接引用样品信息、检测方法、质控批号 | 支持关联项目/样品编号 | 🟡 小 — 关联深度不足 |

### 0.3 关键借鉴点（按优先级排序）

#### P1 — 结构化模板引擎（高优先级）
赛默飞 ELN 的核心竞争力在于**模板驱动的结构化数据录入**，而非自由格式文本。这种模式带来三大优势：
1. **数据可查询**：每个字段都是结构化数据，可直接用于统计分析和报告生成
2. **合规可追溯**：字段级审计追踪，知道"谁在什么时间修改了哪个字段"
3. **减少错误**：通过下拉选择、数值范围校验、必填项控制降低人为错误

**建议 HC-LIMS 补充：**
- 模板设计器：拖拽式字段配置（文本框/数值/下拉/单选/复选/表格/日期/公式）
- 字段校验规则：必填、数值范围、正则表达式、联动显示
- 模板版本管理：模板更新不影响已锁定记录

#### P2 — 计算引擎（高优先级）
赛默飞的公式编辑器支持跨字段引用和常用统计函数。对于高校科研场景（尤其是分析化学实验），计算能力至关重要：
- 浓度计算：C = (A - b) / a（标准曲线法）
- 回收率：R = C_measured / C_spiked × 100%
- RSD 计算：相对标准偏差
- LOD/LOQ 计算：检出限/定量限

**建议 HC-LIMS 补充：**
- 字段间公式计算（如 `result = absorbance * factor + offset`）
- 统计函数库（MEAN, SD, RSD, SLOPE, INTERCEPT, CORREL）
- 单位换算表（mg/L ↔ μg/mL, ppm ↔ ppb）

#### P3 — 仪器数据自动采集（中优先级，依赖 Phase2 仪器集成）
赛默飞通过 Chromeleon™ CDS 实现色谱数据的自动导入，数据直接进入 ELN 对应字段，消除转录错误。

**建议 HC-LIMS 补充（远期）：**
- 仪器原始数据解析器（.txt/.csv/.ms/.raw）
- 峰面积/浓度自动提取并填入 ELN 字段
- 仪器状态与 ELN 记录关联（哪台仪器、何时、谁操作）

#### P4 — 批注与协作（中优先级）
赛默飞支持在记录中直接添加批注，特别适合导师指导学生场景：
- 导师对某个数据点添加质疑批注
- 学生回复说明或修改后重新提交
- 批注线程保留在审计追踪中

**建议 HC-LIMS 补充：**
- 记录级批注面板（类似 Google Docs 评论）
- 批注状态：未读/已读/已解决
- 批注通知：@提及推送消息

#### P5 — 报告导出（中优先级，依赖报告引擎）
赛默飞支持将 ELN 记录一键导出为 PDF 报告，包含签名页、审计追踪摘要、附件清单。

**建议 HC-LIMS 补充：**
- ELN 记录 PDF 导出（含水印、签名页、页码）
- 批量导出（按项目/时间段导出多个记录）
- 报告模板自定义（封面、页眉页脚、Logo）

### 0.4 与 LabWare / StarLIMS ELN 对比

| 特性 | LabWare ELN | StarLIMS ELN | Thermo Fisher ELN | HC-LIMS 目标 |
|-----|-------------|--------------|-------------------|-------------|
| 部署方式 | 本地/私有云 | 本地/SaaS | 本地/云 | 本地/私有云 |
| 模板灵活性 | 极高（LIMS 脚本驱动） | 高（可视化配置） | 中高（字段级配置） | **高（可视化设计器）** |
| 仪器集成 | 极强（200+仪器驱动） | 强（CDS 集成） | **极强（自研 CDS）** | 中（Phase2 扩展） |
| 合规能力 | 21 CFR Part 11 | 21 CFR Part 11 | **21 CFR Part 11 + Annex 11** | 基础电子签名 |
| 适用场景 | 大型药企/CRO | 中型实验室 | **大型研发/制药/临床** | **高校科研/第三方检测** |

**结论：** HC-LIMS ELN 应走"**结构化模板 + 计算引擎 + 电子签名**"路线，避免与赛默飞在仪器集成深度上正面竞争（那是 Phase2 目标），而是在**模板灵活性、科研场景适配、成本优势**上建立差异化。

## 3. 数据模型

### 3.1 ELN 模板 (ELNTemplate)

```typescript
interface ELNTemplate {
  id: string;
  name: string;                    // 模板名称：水质检测原始记录
  category: string;                // 分类：环境检测/食品检测/土壤检测
  version: number;                 // 模板版本号
  status: 'draft' | 'published' | 'archived';
  fields: ELNField[];              // 字段定义
  createdBy: string;
  createdAt: string;
  updatedAt: string;
}

interface ELNField {
  id: string;
  label: string;                   // 显示标签
  type: 'text' | 'number' | 'select' | 'radio' | 'checkbox' | 'table' | 'date' | 'formula' | 'attachment' | 'textarea';
  required: boolean;
  defaultValue?: any;
  options?: string[];              // 下拉/单选/复选选项
  validation?: {                   // 校验规则
    min?: number;                  // 数值最小值
    max?: number;                  // 数值最大值
    regex?: string;                // 正则表达式
    precision?: number;            // 小数位数
  };
  formula?: string;                // 公式表达式，如 "(absorbance - blank) * factor / volume"
  unit?: string;                   // 单位：mg/L, μg/mL
  dependsOn?: string;              // 联动显示条件字段ID
  order: number;                   // 排序序号
}

interface ELNTableColumn {
  id: string;
  label: string;
  type: 'text' | 'number';
  unit?: string;
}
```

### 3.2 ELN 记录 (ELNRecord)

```typescript
interface ELNRecord {
  id: string;
  templateId: string;
  templateName: string;
  templateVersion: number;
  title: string;
  projectName?: string;
  authorId: string;
  authorName: string;
  fields: ELNFieldValue[];         // 字段值快照（锁定后不变）
  content?: string;                // 补充说明（富文本）
  attachments: Attachment[];
  signatures: ELNSignature[];
  annotations: ELNAnnotation[];    // 批注线程
  status: 'draft' | 'submitted' | 'signed' | 'witnessed' | 'locked' | 'returned';
  version: number;                 // 记录版本号
  lockedSnapshot?: string;         // 锁定时的 PDF 快照 URL
  auditTrail: ELNAuditEntry[];     // 审计追踪
  createdAt: string;
  updatedAt: string;
}

interface ELNFieldValue {
  fieldId: string;
  label: string;
  type: string;
  value: any;                      // 字段值
  computed?: any;                  // 公式计算结果
  unit?: string;
}

interface ELNSignature {
  role: 'author' | 'witness' | 'reviewer' | 'qa';
  userId: string;
  userName: string;
  signedAt: string;
  signatureImage?: string;         // 签名图片 base64
  meaning: string;                 // 签名含义："我已审核本记录"
}

interface ELNAnnotation {
  id: string;
  fieldId?: string;                // 关联字段（可选，空则为整篇批注）
  authorId: string;
  authorName: string;
  content: string;
  createdAt: string;
  resolvedAt?: string;
  resolvedBy?: string;
  replies: ELNAnnotationReply[];
}

interface ELNAuditEntry {
  action: 'create' | 'update' | 'lock' | 'witness' | 'return' | 'annotate';
  userId: string;
  userName: string;
  timestamp: string;
  fieldId?: string;                // 字段级变更时记录
  oldValue?: any;
  newValue?: any;
  reason?: string;                 // 变更原因（修改时必填）
}
```

## 4. API 接口

### 4.1 模板管理

```
GET    /api/v1/eln/templates                    # 模板列表（按分类筛选）
POST   /api/v1/eln/templates                    # 创建模板
GET    /api/v1/eln/templates/:id                # 模板详情（含字段定义）
PUT    /api/v1/eln/templates/:id                # 更新模板（草稿状态）
POST   /api/v1/eln/templates/:id/publish        # 发布模板
POST   /api/v1/eln/templates/:id/clone          # 克隆模板（创建新版本）
DELETE /api/v1/eln/templates/:id                # 删除模板（仅草稿）
```

### 4.2 记录管理

```
GET    /api/v1/eln-records                      # 记录列表（筛选+分页）
POST   /api/v1/eln-records                      # 创建记录（基于模板）
GET    /api/v1/eln-records/:id                  # 记录详情
PUT    /api/v1/eln-records/:id                  # 更新草稿
POST   /api/v1/eln-records/:id/submit           # 提交（进入待签名）
POST   /api/v1/eln-records/:id/lock             # 签名锁定
POST   /api/v1/eln-records/:id/witness          # 见证签名
POST   /api/v1/eln-records/:id/return           # 退回修改（带批注）
POST   /api/v1/eln-records/:id/recompute        # 重新计算公式字段
GET    /api/v1/eln-records/:id/versions         # 版本历史
GET    /api/v1/eln-records/:id/pdf              # 导出 PDF
POST   /api/v1/eln-records/:id/annotate         # 添加批注
PUT    /api/v1/eln-records/:id/annotate/:aid    # 回复/解决批注
DELETE /api/v1/eln-records/:id/annotate/:aid    # 删除批注
```

### 4.3 计算引擎（新增）

```
POST   /api/v1/eln/compute                      # 执行公式计算
  Body: { formula: "(A - B) / C * 100", variables: { A: 1.23, B: 0.05, C: 2.0 } }
  Response: { result: 59.0, unit: "%" }

GET    /api/v1/eln/functions                    # 获取支持的函数列表
  Response: ["MEAN", "SD", "RSD", "SLOPE", "INTERCEPT", "CORREL", "MIN", "MAX"]
```

## 6. 合规要求

### 6.1 21 CFR Part 11 对标（远期目标）

赛默飞 ELN 在制药行业广泛应用的核心原因是其严格的合规能力。HC-LIMS 作为高校/第三方检测场景，可逐步向此标准靠拢：

| 合规要求 | 赛默飞实现 | HC-LIMS 现状 | 计划 |
|---------|-----------|-------------|------|
| 唯一用户标识 | LDAP/AD 集成 | 用户名+密码 | Phase2 LDAP |
| 电子签名绑定 | 签名与记录不可分离 | 签名状态字段 | 需增强 |
| 签名含义声明 | 每次签名需选择含义 | 固定含义 | 需补充 |
| 审计追踪完整性 | 删除操作也记录 | 仅记录状态变更 | 需增强 |
| 记录不可篡改 | 锁定后物理隔离 | 逻辑状态控制 | 需增强 |
| 系统验证文档 | IQ/OQ/PQ 全套 | 无 | 远期 |

## 7. 技术实现建议

### 7.1 公式计算引擎

建议采用 **mathjs** 或自定义表达式解析器实现字段级计算：

```typescript
// 公式表达式示例
const formula = "MEAN(table_1.浓度) / standard * 100";
// 解析步骤：
// 1. 词法分析：识别 MEAN, table_1, 浓度, /, standard, *, 100
// 2. 变量替换：从字段值和上下文获取变量值
// 3. 函数执行：调用统计函数库
// 4. 结果返回：{ result: 25.6, unit: "%" }
```

### 7.2 字段级审计追踪存储

建议使用 JSONB 存储字段变更历史，便于查询和 diff：

```json
{
  "auditTrail": [
    { "timestamp": "2025-06-01T10:00:00Z", "user": "张三", "action": "update",
      "fieldId": "f3", "fieldLabel": "检测结果", "oldValue": "25.4", "newValue": "25.6", "reason": "重新计算" },
    { "timestamp": "2025-06-01T11:00:00Z", "user": "张三", "action": "lock" }
  ]
}
```

### 7.3 PDF 快照生成

建议采用 **html2canvas + jsPDF** 或 **Puppeteer** 服务端生成：
- 锁定记录时自动生成 PDF 快照
- 快照包含：记录内容 + 签名页 + 审计追踪摘要 + 页眉页脚
- 快照文件哈希存入区块链/数据库，确保不可篡改
