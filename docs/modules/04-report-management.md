## 2. 用户故事

### US-01: 检测员编制报告
> 作为**检测员**，检测结果完成后，我在报告编辑器中组装样品信息、检测结果和报告模板，生成完整的检测报告。

**验收标准：**
- [ ] 点击"新建报告"选择关联的检测任务
- [ ] 3 栏编辑器: 左侧样品信息/中间检测结果/右侧模板
- [ ] 从检测任务自动带入结果数据
- [ ] 支持手动调整报告排版（添加/删除段落）
- [ ] 保存草稿后可继续编辑

### US-02: 审核员逐项检查
> 作为**审核员**，收到报告审核任务后，我按照 6 项检查清单逐项确认，确保报告质量。

**验收标准：**
- [ ] 审核页面展示 6 项检查清单:
  1. 样品信息完整性
  2. 检测结果准确性
  3. 标准限值引用正确
  4. 计量单位一致性
  5. 签名完整性
  6. 报告格式规范
- [ ] 每项检查通过/不通过选择
- [ ] 不通过项需填写说明
- [ ] 全部通过后可进入下一步

### US-03: 批准人签发报告
> 作为**技术负责人**，审核通过后我进行最终批准签名，系统使用 SignaturePad 记录手写签名。

**验收标准：**
- [ ] 批准页显示报告完整预览
- [ ] 使用 SignaturePad 进行手写签名
- [ ] 签名后报告状态变为"已签发"
- [ ] 已签发报告不可再编辑
- [ ] 自动生成 PDF 版本

### US-04: 查看报告版本历史
> 作为**质量主管**，我需要查看报告的版本历史和批注记录，了解报告的变更过程。

**验收标准：**
- [ ] 版本历史列表（版本号+时间+操作人+变更说明）
- [ ] 支持版本对比（差异高亮）
- [ ] 批注列表（批注人+时间+内容）
- [ ] 支持下载历史版本

## 3. API 接口

```
GET    /api/v1/reports                      # 报告列表
POST   /api/v1/reports                      # 创建报告
GET    /api/v1/reports/:id                  # 报告详情
PUT    /api/v1/reports/:id                  # 更新报告
PUT    /api/v1/reports/:id/submit           # 提交审核
POST   /api/v1/reports/:id/review           # 审核
POST   /api/v1/reports/:id/approve          # 批准签发
POST   /api/v1/reports/:id/sign             # 签名
GET    /api/v1/reports/:id/pdf              # 导出 PDF
GET    /api/v1/reports/:id/versions         # 版本历史
POST   /api/v1/reports/:id/annotations      # 添加批注
```
