## 2. 用户故事

### US-01: 质控员查看控制图
> 作为**质控员**，我定期查看 L-J 控制图，了解质控样品的检测趋势，发现异常及时处置。

**验收标准：**
- [ ] 控制图展示质控样品浓度的时间序列
- [ ] 均值线、±1SD、±2SD、±3SD 控制限清晰标注
- [ ] 超出 ±2SD 的点标记为黄色警告
- [ ] 超出 ±3SD 的点标记为红色失控
- [ ] 违反 Westgard 规则的点自动标注规则名称

### US-02: 质量主管处理偏差
> 作为**质量主管**，发现检测偏差后，我在系统中创建 CAPA 记录，分配责任人跟进整改。

**验收标准：**
- [x] 偏差列表: 编号、来源、标题、状态、责任人、创建时间
- [x] 创建偏差: 填写来源、级别、描述、责任人（点击「新增偏差」）
- [x] 编辑偏差: 弹窗修改状态、级别、CAPA状态、描述、责任人、影响评估
- [x] 5Why 根因分析: 在偏差详情 Drawer 中逐条添加 Why 记录（最多5条）
- [x] 制定 CAPA 措施: 弹窗添加纠正/预防措施，含描述、责任人、完成期限
- [x] CAPA 完成后可标记完成，偏差可关闭

### US-03: 质控员管理质控样品
> 作为**质控员**，我需要维护质控样品台账，记录每次测试并管理过期样品。

**验收标准：**
- [x] 新增质控样品: 弹窗填写名称、批号、标准值、单位、有效期、基体
- [x] 记录测试: 点击「记录测试」增加测试次数
- [x] 删除质控样品: Popconfirm 确认后删除
- [x] 有效期过期自动标红

## 3. API 接口

```
GET    /api/v1/qc/overview                  # 质控总览数据
GET    /api/v1/qc/chart                     # 控制图数据
GET    /api/v1/qc/rules                     # Westgard 规则配置
POST   /api/v1/qc/tests                     # 录入质控结果

GET    /api/v1/qc/deviations                # 偏差列表
POST   /api/v1/qc/deviations                # 创建偏差
GET    /api/v1/qc/deviations/:id            # 偏差详情
POST   /api/v1/qc/deviations/:id/capa       # 添加 CAPA
PUT    /api/v1/qc/capa/:id/verify           # 验证 CAPA
```
