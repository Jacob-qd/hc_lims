## 2. 用户故事

### US-01: 仪器管理员查看台账
> 作为**仪器管理员**，我通过仪器列表了解全部仪器的基本信息和当前状态。

**验收标准：**
- [ ] 仪器列表展示: 名称/编号/型号/状态/部门/校准日期
- [ ] 按运行/空闲/故障/维护中/校准到期 状态筛选
- [ ] 状态标签颜色标识（运行=绿/空闲=蓝/故障=红/维护=橙）
- [ ] 点击仪器行跳转到详情页

### US-02: 检测员查看仪器详情
> 作为**检测员**，使用仪器前查看其校准状态和维护记录，确认可用性。

**验收标准：**
- [ ] 详情页 Tab: 基本信息/校准记录/维护记录/预约/关联方法
- [ ] 基本信息: 仪器图片、名称、型号、序列号、厂家、部门
- [ ] 校准记录: 校准日期/有效期/结果/证书
- [ ] 维护记录: 维护类型/日期/内容/负责人
- [ ] 校准到期前红色预警

### US-03: 维护工程师管理维护计划
> 作为**维护工程师**，我通过维护日历安排预防性维护，记录每次维护内容。

**验收标准：**
- [ ] 维护日历: 月视图，标记维护事件
- [ ] 标记颜色: 计划(蓝)/执行中(绿)/逾期(红)
- [ ] 点击事件查看维护详情
- [ ] 维护完成后状态更新

## 3. API 接口

```
GET    /api/v1/instruments                   # 仪器列表
POST   /api/v1/instruments                   # 新增仪器
GET    /api/v1/instruments/:id               # 仪器详情
PUT    /api/v1/instruments/:id               # 更新仪器
DELETE /api/v1/instruments/:id               # 删除仪器

GET    /api/v1/instruments/:id/calibrations  # 校准记录
POST   /api/v1/instruments/:id/calibrations  # 新增校准
GET    /api/v1/instruments/:id/maintenance   # 维护记录
POST   /api/v1/instruments/:id/maintenance   # 新增维护

GET    /api/v1/instruments/calendar          # 维护日历
GET    /api/v1/instruments/statistics        # 状态分布统计
```
