## 2. 用户故事

### US-01: 库管员入库试剂
> 作为**库管员**，收到采购的试剂后，在系统中登记入库信息，更新库存数量。

**验收标准：**
- [ ] 点击"入库"弹出入库弹窗
- [ ] 选择或搜索品名，填写数量、批号、有效期、供应商
- [ ] 提交后库存数量自动更新
- [ ] 生成入库记录

### US-02: 检测员查看临期试剂
> 作为**检测员**，使用试剂前查看其有效期，确保不使用过期试剂。

**验收标准：**
- [ ] 列表页临期（30天内到期）行红色标记
- [ ] 已过期试剂灰色显示并标记"已过期"
- [ ] 排序: 按到期日降序

### US-03: 实验室主任发起采购申请
> 作为**实验室主任**，发现试剂库存不足时，在系统中发起采购申请。

**验收标准：**
- [x] 点击「采购申请」弹窗填写物料、规格、数量、单价、紧急程度
- [x] 提交后采购申请列表实时更新
- [x] 可查看采购申请详情（单号、申请人、部门、物料明细）

### US-04: 库管员管理供应商
> 作为**库管员**，我需要维护供应商信息，方便采购时选择。

**验收标准：**
- [x] 供应商列表: 名称、联系人、电话、邮箱、评级、状态
- [x] 新增供应商: 弹窗填写名称、联系人、电话、邮箱、评级、状态
- [x] 编辑供应商: 弹窗修改已有供应商信息
- [x] 删除供应商: Popconfirm 确认后删除

### US-05: 库管员执行出入库与盘点
> 作为**库管员**，我需要准确记录每次出入库和盘点，确保账实相符。

**验收标准：**
- [x] 入库: 弹窗填写试剂信息，提交后库存列表新增条目，自动生成入库记录
- [x] 出库: 列表单选一行，弹窗填写出库数量，提交后库存扣减，自动生成出库记录
- [x] 盘点: 列表单选一行，弹窗填写实盘数量，系统自动计算差异，更新库存，生成盘点记录
- [x] 出入库记录 Tab 展示全部变动历史

## 3. API 接口

```
GET    /api/v1/inventory                     # 库存列表
POST   /api/v1/inventory/inbound             # 入库
POST   /api/v1/inventory/outbound            # 出库
GET    /api/v1/inventory/alerts              # 预警列表
POST   /api/v1/inventory/purchase-request    # 采购申请
GET    /api/v1/inventory/purchase-requests   # 采购申请列表
```
