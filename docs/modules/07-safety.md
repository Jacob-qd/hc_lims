## 1. 功能概述

高校实验室安全管理模块，覆盖危化品全生命周期管理、日常安全检查、废弃物处置追踪三大核心场景。

### 已实现功能

| 功能点 | 状态 | 说明 |
|--------|:----:|------|
| 危化品台账 | ✅ | 名称/CAS/分类/存量/位置/责任人/MSDS/状态 |
| 存量预警 | ✅ | 库存不足/过期预警状态标记 |
| 安全检查清单 | ✅ | 日检项目列表（通风/喷淋/灭火器/钢瓶固定） |
| 废弃物管理 | ✅ | 废液/固废分类暂存、产生日期、转运状态 |
| 新增化学品 | ✅ | 弹窗录入，提交后更新台账 |
| KPI 统计 | ✅ | 化学品总数/存量预警/待处理废弃物 |

## 2. User Stories

### US-01: 实验室管理员登记危化品
> 作为**实验室管理员**，我采购危化品后在系统中登记入库信息，以便追踪库存和安全状态。

**验收标准：**
- [x] 录入化学品名称、CAS号、危险分类、存量、存放位置
- [x] 标记MSDS是否齐全
- [x] 自动分配责任人
- [x] 入库后状态为"正常"

### US-02: 实验员查看危化品存量预警
> 作为**实验员**，我在实验前查看危化品存量，避免实验中途发现试剂不足。

**验收标准：**
- [x] 台账列表显示所有化学品当前存量
- [x] 存量不足时状态变为"预警"
- [x] 支持按名称/CAS号搜索

### US-03: 安全员执行日检
> 作为**安全员**，我每天按照检查清单巡查实验室，确认安全设施正常。

**验收标准：**
- [x] 系统提供标准日检清单（通风/喷淋/灭火器/钢瓶固定/废弃物标识）
- [x] 逐项确认后标记"符合"
- [x] 不符合项记录并触发整改通知

### US-04: 废弃物分类暂存与转运
> 作为**实验室管理员**，我按规定分类收集实验废弃物，记录暂存信息并安排合规转运。

**验收标准：**
- [x] 按类型分类（有机废液/重金属废液/废弃试剂瓶/固废）
- [x] 记录来源实验、数量、暂存位置、产生日期
- [x] 状态流转：暂存中 → 待转运 → 已转运

## 3. 数据模型

```typescript
interface Chemical {
  id: string;
  name: string;           // 化学品名称
  cas?: string;           // CAS号
  category: string;       // 危险分类（剧毒/易燃/腐蚀/氧化/一般）
  stock: string;          // 存量（如 "5L", "500g"）
  location: string;       // 存放位置
  responsible: string;    // 责任人
  msds: string;           // MSDS 状态（有/无）
  status: 'normal' | 'warning';  // 正常 / 预警
  purchaseDate?: string;
  expiryDate?: string;
}

interface WasteRecord {
  id: string;
  type: string;           // 废弃物类型
  source: string;         // 来源实验
  amount: string;         // 数量
  location: string;       // 暂存位置
  status: 'temporary' | 'pending' | 'transferred';
  date: string;
  transferredBy?: string;
  transferredAt?: string;
}

interface SafetyCheckItem {
  id: string;
  item: string;           // 检查项名称
  category: string;       // 类别（通风/消防/危化品/电气）
  frequency: string;      // 检查频率（日/周/月）
  status: 'pass' | 'fail' | 'pending';
  checkedAt?: string;
  checkedBy?: string;
  remark?: string;
}
```

## 4. API 接口

```
GET    /api/v1/research/chemicals              # 危化品列表
POST   /api/v1/research/chemicals              # 新增化学品
PUT    /api/v1/research/chemicals/:id          # 更新化学品信息
DELETE /api/v1/research/chemicals/:id          # 删除记录

GET    /api/v1/research/waste-records          # 废弃物记录
POST   /api/v1/research/waste-records          # 新增废弃物记录
PUT    /api/v1/research/waste-records/:id      # 更新状态（转运）

GET    /api/v1/research/safety-checks          # 安全检查清单
POST   /api/v1/research/safety-checks/:id      # 提交检查结果
```

## 6. 与高校科研版其他模块的关联

| 关联模块 | 数据流向 | 说明 |
|---------|---------|------|
| 检测管理 | 实验 → 废弃物 | 检测实验产生的废液自动归集到废弃物台账 |
| 仪器管理 | 仪器维护 → 化学品消耗 | 仪器维护使用的试剂自动扣减库存 |
| 教学实验 | 教学 → 化学品申领 | 教学实验课申领危化品，审批后出库 |
| 库存管理 | 化学品 → 库存同步 | 危化品库存与通用库存双向同步 |
