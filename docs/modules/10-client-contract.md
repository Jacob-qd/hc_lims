### 用户故事

### US-01: 业务员管理客户信息
> 作为**业务员**，我在客户档案中维护客户信息，查看客户的委托历史和合同执行情况。

**验收标准：**
- [ ] 客户列表: 名称/类型/联系人/电话/状态/合同数
- [ ] 按客户类型（企业/政府/高校/个人）筛选
- [ ] 详情页: 基本信息 Tab + 委托项目 Tab + 合同 Tab

### US-02: 合同管理员跟踪合同
> 作为**合同管理员**，我管理检测委托合同，跟踪合同的执行状态和回款情况。

**验收标准：**
- [ ] 合同列表: 编号/客户/金额/签订日期/到期日期/状态
- [ ] 状态标签: 执行中/已完成/已终止/待续签
- [ ] 到期前 30 天提醒续签

### API

```
GET    /api/v1/clients                      # 客户列表
POST   /api/v1/clients                      # 新增客户
GET    /api/v1/clients/:id                  # 客户详情
PUT    /api/v1/clients/:id                  # 更新客户

GET    /api/v1/contracts                    # 合同列表
POST   /api/v1/contracts                    # 新增合同
GET    /api/v1/contracts/:id                # 合同详情
PUT    /api/v1/contracts/:id                # 更新合同
```
