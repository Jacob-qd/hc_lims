## 主流程 A：报告列表浏览

```
[报告管理] → [加载报告列表]
    → KPI卡片: 待撰写/待技术审核/待批准签发/已签发
    → 筛选: 编号/样品号/项目名称/类型/状态/日期
    → 表格展示报告列表
    → 点击行 → 右侧详情抽屉 (报告信息/流程步骤条/PDF预览)
    → 批量操作: 生成报告/提交审核/批准签发/下载PDF
```

## 主流程 B：报告编制

```
[报告列表 → 点击"生成报告"或"编辑"] → [报告编辑器]
    → 左侧: 报告结构导航 (封面/基本信息/样品信息/检测结果/结论/审核记录/附件)
    → 中间: 编辑区
        → 封面: 公司名/编号/委托单位/项目名称/样品类型/采样地点/日期
        → 检测结果: 表格 (序号/项目/单位/结果/检出限/限值/方法/判定)
        → 编制/审核/批准签名区 + 电子签章
    → 右侧: 流程状态面板 + 批注功能 + 变更历史
    → 操作: 保存/生成PDF/提交审核
```

## 主流程 C：结果审核

```
[报告 → 提交审核] → [审核弹窗弹出]
    → 审核结论: 通过/不通过
    → 审核意见: 文本框
    → 审核要点检查清单 (6项: 样品信息/检测项目/仪器校准/质控/数据/编制规范)
    → 每项勾选: 符合/不符合
    → 确认 → POST /api/v1/reports/:id/review
    → 状态更新
```

## 主流程 D：报告状态流转

```
[草稿] --提交审核--> [待技术审核]
[待技术审核] --通过--> [待批准签发] | --退回--> [草稿]
[待批准签发] --批准--> [已签发] | --退回--> [待技术审核]
[已签发] --归档--> [已归档]
[已签发] --撤回--> [草稿] (需权限)
```

## 交互流程

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 筛选报告 | 表格实时过滤 |
| 2 | 点击行 | 右侧抽屉展开 (流程步骤条+PDF预览) |
| 3 | 点击"编辑" | 进入报告编辑器 |
| 4 | 左侧导航点击章节 | 中间区切换编辑内容 |
| 5 | 编辑检测结果表格 | 自动计算判定 (符合/不符合) |
| 6 | 点击"提交审核" | 审核弹窗 |
| 7 | 勾选审核要点清单 | 6项全部通过方可提交 |
| 8 | 输入审核意见 | 非必填 |
| 9 | 确认提交 | POST → 状态更新 |
