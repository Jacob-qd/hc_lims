# P1-03 批量操作增强

> 对标: LabWare 批量分配/签发/导入 | StarLIMS Batch Processing
> 版本: v1.0 | 优先级: P1 | 估算: 2周

---

## 1. 功能概述

为各模块增加批量操作能力，解决当前逐条处理的效率瓶颈。

### 涉及模块

| 模块 | 新增批量操作 | 当前状态 |
|------|------------|:--------:|
| 样品管理 | 批量导入(Excel)、快速复制 | ⚠️ 占位按钮 |
| 检测管理 | 批量分配、批量排期 | ❌ 逐个分配 |
| 报告管理 | 批量签发、批量导出PDF | ❌ 逐份签发 |
| 库存管理 | 批量入库、Excel导入 | ❌ 逐个录入 |
| 仪器管理 | 批量校准登记 | ❌ 逐个登记 |

---

## 2. 功能需求

### FR-01: Excel 批量导入

**流程:**
```
选择业务类型 → 下载模板 → 填写数据 → 
上传文件 → 服务端校验 → 
结果预览（成功/失败明细）→ 确认提交
```

**校验规则:**
- 必填字段检查
- 数据格式校验（日期、数字、枚举值）
- 业务规则校验（如样品编号唯一性）
- 关联数据校验（如客户 ID 有效性）
- 错误定位: 行号+列号+错误描述

### FR-02: 多选批量处理

| 操作 | 选择范围 | 处理方式 |
|------|---------|---------|
| 批量分配 | 多个检测任务 | 统一指定检测人/排期 |
| 批量签发 | 多份已审核报告 | 统一签发，记录批量操作人 |
| 批量导出 | 多份报告 | 打包为 ZIP，组织文件夹结构 |
| 批量入库 | 多批试剂 | 统一填写入库信息 |
| 批量校准 | 多台仪器 | 统一登记校准日期/结果 |

### FR-03: 快速复制

- 基于已有样品创建新样品
- 复制内容: 客户信息、检测项目、部分字段
- 新的样品编号自动生成
- 适用于重复性检测场景

## 3. 技术方案

```
┌────────────────────────────────────┐
│    通用批量操作框架                  │
├────────────────────────────────────┤
│                                    │
│  BatchActionBar (React 组件)        │
│  ┌──────────────────────────────┐  │
│  │ 多选 N 项 → [批量导入]       │  │
│  │             [批量分配]       │  │
│  │             [批量签发]       │  │
│  └──────────────────────────────┘  │
│                                    │
│  BatchImportModal                   │
│  ┌──────────────────────────────┐  │
│  │ Step1: 下载模板               │  │
│  │ Step2: 上传文件               │  │
│  │ Step3: 校验预览               │  │
│  │ Step4: 确认提交               │  │
│  └──────────────────────────────┘  │
│                                    │
│  BatchResultDisplay                 │
│  ┌──────────────────────────────┐  │
│  │ 成功: 45/50                   │  │
│  │ 失败: 5                      │  │
│  │  → 第3行: 日期格式错误        │  │
│  │  → 第17行: 客户不存在         │  │
│  └──────────────────────────────┘  │
└────────────────────────────────────┘
```

### 后端

- Excel 解析: `xlsx` (SheetJS) 库
- 校验引擎: 按业务规则链式校验
- 事务提交: 全部成功或全部回滚（保证数据一致性）
- 异步处理: 大批量导入后台执行，WebSocket 推送进度
