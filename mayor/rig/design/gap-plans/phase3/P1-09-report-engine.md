# P1-09 报表引擎

> 对标: StarLIMS StarStation | LabWare Crystal Reports
> 版本: v1.0 | 优先级: P1 | 估算: 8周

---

## 1. 功能概述

实现自定义报表系统，支持可视化报表设计、预置报表模板、定时调度和自动分发。

### 核心能力

- 嵌入式 BI 仪表盘（集成 MetaBase）
- 预置报表模板（样品/检测/QC/仪器/人员）
- 可视化报表设计器（拖拽式）
- 定时调度 + 自动分发
- 导出格式: PDF / Excel / HTML / 图像

---

## 2. 用户故事

### US-01: 实验室主任查看月度 KPI
> 作为**实验室主任**，每月初自动收到上月检测量、及时率、超标率的统计报表。

### US-02: 质量主管自定义报表
> 作为**质量主管**，我可以拖拽创建需要的报表，选择数据源和图表类型。

### US-03: 检测员查看工作量
> 作为**检测员**，我可以查看个人工作统计报表，了解自己的任务完成情况。

---

## 3. 功能需求

### FR-01: 预置报表模板（10+）

| 分类 | 报表名称 | 数据源 | 图表类型 |
|------|---------|--------|---------|
| KPI | 实验室月度运营报告 | 样品/检测/报告 | 指标卡片+趋势图 |
| 样品 | 样品量趋势（按类型/部门/时间） | 样品表 | 柱状图+折线图 |
| 样品 | 样品及时率统计 | 样品+检测表 | 饼图+趋势图 |
| 检测 | 检测项目分布 | 检测表 | 环形图 |
| 检测 | 仪器检测量排名 | 检测+仪器表 | 柱状图（Top 10） |
| QC | 质控月度报告 | QC 测试表 | 控制图+汇总表 |
| QC | 超标/异常统计 | QC 规则判定表 | 趋势图+明细表 |
| 仪器 | 仪器利用率报告 | 仪器+使用记录表 | 仪表盘+趋势图 |
| 人员 | 人员工作量统计 | 任务表 | 柱状图+排名表 |
| 综合 | 部门绩效对比 | 各业务表 | 雷达图+柱状图 |

### FR-02: 嵌入式 BI 方案

推荐: **MetaBase** 嵌入式集成

```
┌────────────────────────────────────────────┐
│             HC-LIMS UI                     │
│  ┌──────────────────────────────────────┐ │
│  │  报表中心 (React)                    │ │
│  │  · 报表列表                          │ │
│  │  · 参数选择                          │ │
│  │  · MetaBase iframe 嵌入              │ │
│  └──────┬───────────────────────────────┘ │
└─────────┼──────────────────────────────────┘
          │ Embedding API
┌─────────┼──────────────────────────────────┐
│         ▼                                   │
│  ┌──────────────────────────────────────┐   │
│  │          MetaBase Server              │   │
│  │  · 仪表盘设计器                      │   │
│  │  · SQL 查询编辑器                    │   │
│  │  · 定时刷新                          │   │
│  │  · 邮件发送                          │   │
│  │  · 嵌入认证 (JWT SSO)                │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  部署: Docker 单独容器                      │
│  数据库: 直接连接 HC-LIMS PostgreSQL        │
└─────────────────────────────────────────────┘
```

### FR-03: 备选方案 — 自研轻量版

若 MetaBase 集成不合适，自研方案:

| 组件 | 选型 | 说明 |
|------|------|------|
| 图表 | `@ant-design/charts` (AntV) | 与现有 AntD 整合 |
| 报表布局 | `react-grid-layout` | 拖拽布局 |
| 参数配置 | JSON Schema | 与动态表单统一 |
| 数据查询 | 后端 REST API | SQL → 聚合 → 返回 |
| PDF 导出 | `html2canvas` + `jspdf` | 截图转 PDF |
| 定时调度 | `node-cron` | 服务端 cron 任务 |
| 自动分发 | Nodemailer / 企业微信 | 邮件/消息推送 |

## 4. API 设计

```
# 报表模板
GET    /api/v1/reports/templates                 # 模板列表
POST   /api/v1/reports/templates                 # 创建模板
PUT    /api/v1/reports/templates/:id              # 更新模板
DELETE /api/v1/reports/templates/:id              # 删除模板

# 报表生成
POST   /api/v1/reports/generate                  # 生成报表
GET    /api/v1/reports/generated                  # 已生成的报表列表
GET    /api/v1/reports/generated/:id/download     # 下载报表

# 调度
GET    /api/v1/reports/schedules                  # 调度配置
POST   /api/v1/reports/schedules                  # 创建调度
PUT    /api/v1/reports/schedules/:id               # 更新调度

# MetaBase 集成 (备选)
GET    /api/v1/metabase/embed/:dashboardId        # 获取嵌入 URL
POST   /api/v1/metabase/sso                       # SSO 令牌
```
